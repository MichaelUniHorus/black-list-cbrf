# Open Black - Общественная карта компаний с признаками нелегальной деятельности

> **⚠️ ВАЖНО: Это общественный проект на добровольных началах**  
> Цель проекта - защитить граждан России от компаний с выявленными признаками нелегальной деятельности на финансовом рынке путем визуализации официального списка Центрального Банка РФ.

## 🛡️ Миссия проекта

Каждый день тысячи людей становятся жертвами компаний с признаками нелегальной деятельности на финансовом рынке. **Open Black** - это не просто карта, это **инструмент общественной защиты**, который помогает:

- 👨‍👩‍👧‍👦 **Гражданам** - избежать обращения в компании с выявленными признаками нелегальной деятельности
- 🏛️ **Обществу** - повысить осведомленность о финансовых рисках
- 📰 **СМИ и активистам** - получить данные для расследований
- 🎓 **Исследователям** - анализировать распространение нелегальной деятельности на финансовом рынке

## 💡 Почему это важно

- 📈 **Тысячи организаций** в черном списке ЦБ РФ
- 🌍 **Филиалы по всей стране** - часто скрытые от обычного поиска
- 💰 **Миллиарды рублей** потерь граждан ежегодно
- ⚖️ **Публичный доступ** к официальным данным в удобном формате

**Open Black делает невидимое видимым** - показывая на карте, где именно находятся опасные организации.

## 🎯 Основные возможности

- 🗺️ **Интерактивная карта** - визуализация компаний с выявленными признаками нелегальной деятельности
- 🔍 **Автоматический поиск филиалов** - через Yandex Maps API
- 📊 **Статистика и аналитика** - по регионам и категориям нарушений
- 🔄 **Ежедневное обновление** - автоматическая синхронизация со списком ЦБ РФ
- 🌐 **Открытый API** - данные доступны для интеграции в другие проекты
- 🆓 **Полностью бесплатно** - проект создан волонтерами для общества

## 🏗️ Технологический стек

### Backend
- Python 3.11+ + FastAPI
- SQLAlchemy ORM (SQLite для dev, PostgreSQL для prod)
- Celery + Redis для фоновых задач
- Flower для мониторинга Celery
- Alembic для миграций БД

### Frontend
- React + TypeScript + Vite
- Yandex Maps API
- TailwindCSS + shadcn/ui
- Axios для API запросов

### Интеграции
- ЦБ РФ API (JSON черный список)
- Yandex Maps API (геокодинг и поиск организаций)

## 📋 Требования

- Python 3.11+
- Node.js 18+
- Redis 6+
- PostgreSQL 14+ (для production)

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd open-black
```

### 2. Получение API ключей

Следуйте инструкциям в файле [`API_KEYS_GUIDE.md`](./API_KEYS_GUIDE.md) для получения:
- ✅ Yandex Maps API ключ (бесплатно до 25,000 запросов/день)

### 3. Настройка окружения

```bash
# Скопируйте пример .env файла
cp .env.example .env

# Отредактируйте .env и добавьте ваш Yandex Maps API ключ
# YANDEX_MAPS_API_KEY=ваш-ключ-здесь
```

### 4. Запуск через Docker Compose (рекомендуется)

```bash
# Запуск всех сервисов (backend, frontend, redis, postgres)
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановка
docker-compose down
```

Сервисы будут доступны по адресам:
- Frontend: http://localhost:5173
- Backend API: http://localhost:8000
- API Docs: http://localhost:8000/docs
- Flower (мониторинг): http://localhost:5555

### 5. Ручная установка (для разработки)

#### Backend

```bash
cd backend

# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Установка зависимостей
pip install -r requirements.txt

# Запуск миграций
alembic upgrade head

# Запуск сервера
uvicorn app.main:app --reload --port 8000

# В отдельном терминале: запуск Celery worker
celery -A app.tasks.celery_app worker --loglevel=info

# В отдельном терминале: запуск Celery Beat (планировщик)
celery -A app.tasks.celery_app beat --loglevel=info

# В отдельном терминале: запуск Flower
celery -A app.tasks.celery_app flower --port=5555
```

#### Frontend

```bash
cd frontend

# Установка зависимостей
npm install

# Запуск dev сервера
npm run dev
```

## 📚 API Документация

После запуска backend сервера документация доступна по адресам:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

### Основные endpoints

- `GET /api/organizations` - список организаций (пагинация, фильтры)
- `GET /api/organizations/{id}` - детали организации
- `GET /api/locations` - все точки для карты
- `GET /api/search` - поиск по названию/ИНН
- `GET /api/stats` - статистика
- `POST /api/admin/sync-cbr` - синхронизация с ЦБ РФ
- `POST /api/admin/enrich/{id}` - обогащение данных организации

## 🌸 Мониторинг Celery через Flower

Flower предоставляет веб-интерфейс для мониторинга Celery задач:

- URL: http://localhost:5555
- Логин/пароль: указаны в `.env` (`FLOWER_BASIC_AUTH`)

Возможности:
- Просмотр активных задач
- История выполнения
- Статистика workers
- Графики производительности

## 🗄️ База данных

### Миграции

```bash
# Создать новую миграцию
alembic revision --autogenerate -m "описание изменений"

# Применить миграции
alembic upgrade head

# Откатить миграцию
alembic downgrade -1
```

### Модели данных

- **Organization** - организации из черного списка ЦБ
- **Location** - найденные адреса и координаты
- **EnrichmentTask** - задачи обогащения данных

## 🔄 Фоновые задачи

### Celery задачи

1. **enrich_organization_addresses** - поиск адресов через Yandex Maps
2. **periodic_cbr_sync** - ежедневная синхронизация с ЦБ РФ (запускается автоматически)

### Ручной запуск задач

```python
from app.tasks import enrich_organization_addresses, sync_cbr_data

# Обогатить данные организации
task = enrich_organization_addresses.delay(organization_id=123)

# Синхронизировать с ЦБ
task = sync_cbr_data.delay()

# Проверить статус задачи
print(task.status)
```
